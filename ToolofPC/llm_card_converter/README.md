# LLM Project 角色卡转换工具（PC 端）

把 SillyTavern / TavernAI 角色卡（PNG / JSON）转换为 LLM Project 角色卡
（`.llmchar.png` / `.llmcard`）的独立桌面工具。

> 与主项目（聊天 App）分离：主 App 只做"导入时的轻量转译 + 最终预览"；
> 本工具是重型工作台，支持三步流水线、逐步对照预览、手动修改、单步重跑、批量。

## 目录结构

```text
lib/
├── main.dart            # 入口（界面建设中）
├── core/                # 转换内核（纯 Dart，与主项目共享逻辑的副本）
│   ├── asset_magic.dart
│   ├── conversion_models.dart
│   ├── png_chara_reader.dart
│   ├── third_party_card_detector.dart
│   ├── character_card_mapper.dart      # 第一步：规则转译
│   ├── conversion_service.dart
│   └── conversion_writer.dart          # 输出 .llmchar.png / .llmcard + 报告
├── pipeline/
│   └── pipeline.dart    # 三步流水线编排（每步可单独跑/重跑，保留中间结果）
└── ui/                  # PC 工作台界面（待建）
```

## 三步流水线

1. **规则转译**（离线，已实现）：字段映射 / PNG 元数据读取 / 内嵌世界书。
2. **AI 智能归类**（需配置 API，已实现）：把 description 等自由文本拆分归位到
   姓名 / 关系 / 身体 / 心理 / 背景 / 场景 / 自定义条目；严格防幻觉
   （只重组原文、禁止新增设定）；结构化 JSON 输出，解析/校验失败自动回退规则结果。
   保留示例对话 / 来源信息 / 作者备注 / 原始数据等条目不动。
3. **检查精修**（需 API，已实现）：AI 对照原文审计整理结果，找出漏转 / 疑似新增
   （幻觉）/ 重复归类，产出"问题清单"展示在预览下方（仅供参考，不自动改）。
   NSFW 容错：模型拒绝 / 返回空时视为"无问题"，流程照常完成。

## 卡类型（人物卡 / 系统卡）

LLM Project 角色卡分人物卡与系统卡，二者结构不同。酒馆卡本身无此区分，故：

- **未配置 AI**：一律按人物卡转译（保底）。
- **已配置 AI**：转译前先由 AI 判定类型（第0步）。
    - 人物卡：走人物卡结构（姓名/关系/身体/心理/背景…）+ AI 归类 + 检查精修。
    - 系统卡：走系统卡结构（系统名称/概要/详情{世界设定/世界观/系统机制}/主角设定/剧情），
      自带角色归入自定义条目。

系统卡的 AI 归类会把世界设定 / 世界观 / 系统机制拆分归位，自带角色拆为自定义条目。
第三步会复核类型，若判定不符，自动切换类型并重跑一次（带防死循环上限）。

## 转译（单张 / 批量统一）

主页选择文件后会进入待转列表（支持「选择文件」多选、「文件夹」递归扫描、
拖入多个文件/文件夹），可在主页先检查、增删，再点「开始转译」。

工作区统一处理 1~N 张：

- 逐张**串行**转译（避免 API 限流）。
- 左侧两个进度条：**整体进度**（第几张/共几张）+ **当前卡片进度**（三步走到哪），
  并有逐张滚动日志。
- 中间卡片列表：**正在转译的自动展开**显示预览，其余折叠成栏目；完成后可随意
  展开任意一张查看 / 编辑 / 比对。
- 右上「AI」开关可整体关闭 AI（只跑规则转译，更快/省费用）；转译中不可切换。
- 单张失败不影响其他张。
- 「完成并保存」一键把成功的卡导出到默认转译位置（已保存的标记绿色）。

## 自定义标题栏

桌面端隐藏系统标题栏，改用应用内自绘标题栏（顶部 40px）：可拖动移动窗口、
双击最大化/还原、右侧最小化 / 最大化 / 关闭按钮。基于 `window_manager`。

> 若升级后窗口异常（无法拖动/缩放），在项目目录执行 `flutter clean && flutter pub get`
> 重新生成原生工程即可。

## 历史记录

主页右侧展示转译历史（轻量清单，不保存转换结果本身）。每张卡只要转译过就记一条：

- 角色名、转译时间、封面缩略图（有立绘时）
- 两套独立标签：**成功 / 失败**、**已保存 / 未保存**
- 已保存的显示保存路径
- 可逐条删除，或一键清空（不影响已导出的角色卡文件）

## 版本比对

工作区展开某张卡后点「比对」进入独占的比对页（只读），横向并排展示：
**原文 → 规则转译 → AI 智能归类 → 成品** 四个版本，便于核对每一步的变化与
信息忠实度。各列统一为卡片样式；原文列的开场白、示例对话、内嵌世界书、元信息
默认折叠（点开查看），与右侧版本观感对齐。

## 手动编辑

工作区右侧「编辑」可逐条修改角色设定条目：结构化条目（姓名/身体/心理/背景等）
按子字段编辑，纯文本条目整段编辑，并可切换每条的启用状态。保存后，预览中
**被改动的条目以绿色高亮**标出，便于核对（配合第三步审计建议进行修正）。

## 预览

结果预览中，角色设定条目**可点击展开**查看完整内容（结构化条目按子字段展开），
便于核对 AI 归类是否正确。「简短描述」由 AI 生成的一句话概括填充（原始长描述
仍保留在背景条目与原始数据中）。

## AI 配置

主页「设置」→「AI 配置」：选预设厂家（自动填 Base URL）→ 填 API Key →
「获取模型」自动拉取可用模型 → 选模型 → 保存。未配置时第二步自动跳过。

每一步都可：对照预览（上一步 vs 本步）、手动修改、单独重跑。
批量转译后可逐张检查每张卡的三步过程。

## 保存位置

- 首次「完成」时询问保存目录，并记住为「默认转译位置」；之后直接保存、不再询问
  （批量转译也不逐次询问）。
- 工作区「另存为…」可单次指定其他目录（并更新默认）。
- 主页「设置」可查看 / 修改 / 清除默认位置。
- 不强制保存到 C 盘 / Downloads，由用户自选。

  的"导入即转译"功能。
