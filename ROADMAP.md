# Roadmap

本文档记录 LLM Project 当前阶段的开发计划。  
项目仍处于 Demo 阶段，规划内容可能根据实际体验、用户反馈和技术实现难度调整。

---

## 版本规则

### Demo 阶段

当前项目处于 Demo 阶段。

```text
Demo 1.x.x
```

用于构建正式版 `v1.0.0` 所需的基础功能。

### 正式版阶段

正式版发布后采用：

```text
vX.Y.Z
```

含义：

- `X.0.0`：某一大方向完成
- `X.Y.0`：通往下一个大方向的重要工程
- `X.Y.Z`：该工程内的优化、修复和细分版本

## Demo 1.2.7：第三方卡导入 + 界面更新版（当前主攻）

优先级：P0（先于 1.3.x）

本版聚焦三件事：让用户能用上更多角色卡、换上更清晰的界面、
并让角色状态变得可见可用。

### 主要内容

1. **导入第三方角色卡（转换工具移动端整合）**
  - 规则转换核心、AI 智能归类、AI 检查精修 + 源文对照（005-A/B/C）已完成。
  - 本版将其整合进移动端：「导入角色卡」自动识别第三方格式并复用同一套核心。
  - 可直接导入主流酒馆 / Character Card 角色卡的设定、世界书与开场白。
2. **界面更新（UI 大改）**
  - 整体视觉与排版优化，提升清晰度与一致性。
3. **状态栏基础版**
  - 让角色当前状态（好感度 / 心情 / 时间 / 地点等）可见，并可注入 Prompt。

### 底层改造：会话副本 Prompt 机制

为支撑状态栏注入与后续的动态设定演化，本版引入「会话副本」机制：

- 角色卡的原始 system prompt 作为「母版」，运行时不会被改写。
- 进入会话时，从母版克隆出一份「会话副本」，实际注入使用副本。
- 会话副本可被状态栏数值、界面交互（及后续的自动更新）修改。
- 清空聊天记录时，副本被删除并从母版重新克隆，恢复到初始设定。
- 复用已上线的 `{{char}} / {{user}}` 统一渲染，在其基础上支持来自副本的动态变量。

该机制是状态栏注入、界面交互影响 Prompt、未来动态设定演化的共同基础。

### 界面交互样板（预览）

本版加入一个可玩的交互样板，用于验证「界面交互影响 Prompt」的完整链路：

- 在开场白中放置交互控件（如输入主角姓名、选择技能、换一批、确认）。
- 确认后写入会话副本，真正影响后续对话的设定，清空记录后可还原。

本版仅提供样板，可视化「积木式 UI」平台化能力将在后续版本逐步推进
（见文末「声明式 UI 积木系统」）。

优先级：P3

### 设定版本管理

- 手动保存 / 命名 / 备注角色设定版本
- 查看版本列表、恢复指定版本

### 自动设定维护

- LLM 定期总结对话，生成新版设定草稿（先存草稿，用户确认后再应用）
- 启用但为空的条目可由 LLM 自动填充
- 自动更新不会覆盖原始设定，借助会话副本机制，修改可以还原

目标：

让 LLM 辅助维护设定但不自动覆盖原始设定，且修改可回退。

## 声明式 UI 积木系统（平台级，多版本推进）

定位：从 1.2.7 的交互样板，逐步演化为可拼装的可视化界面系统。
这是一个跨多个版本推进的方向，不会在单一版本内一次性完成。

分层架构（关键设计）：

- **第 1 层：原子积木（原语）**
  - 进度条 / 标签行 / 按钮组 / 文本块 / 文本输入 / 图片槽 / 折叠面板 / 状态点 等。
  - 用 Dart 实现，随版本发布逐步增加种类。
  - 起步先做能覆盖大多数卡的 6~8 个基础原语，再逐步扩展。
- **第 2 层：复合模块 / 模板（资产）**
  - 用第 1 层原语组合 + 字段绑定 + 样式参数，以纯数据（JSON）描述。
  - 可由转换工具生成，或用户导入入库，支持更新与分享。
  - 仅以数据描述，不下发可执行代码：移动端无法在运行时编译执行外部代码，也避免第三方代码带来的安全风险。
- **引擎**：App 内置「积木解释引擎」，读第 2 层数据 → 调第 1 层原语渲染（思路同官方 RFW）。

交互影响 Prompt：

- 界面交互（输入 / 选择等）写入会话副本 Prompt，经渲染注入对话，清空记录后还原。
- 依赖 1.2.7 引入的会话副本机制。

转换映射（较复杂，靠后推进）：

- 第三方卡的正则脚本 + 世界书「输出格式」实际描述的是一份数据约定（有哪些字段 / 选项 / 交互），
  其中的 CSS / HTML 只是外观皮肤。
- 转换的目标是：读懂数据约定，舍弃具体皮肤，再决定复用已有模板或用原语组合出新模板。
- 能力边界：
  - 🟢 结构化数据面板（状态栏 / 属性 / 选项 / 档案）：可达 ~90%。
  - 🟡 仿真界面（聊天气泡 / 立绘切换 / 仿社交 App）：可达 ~60-75%，精细动画 / 像素级皮肤做不到。
  - 🔴 JS 驱动游戏逻辑（MVU / tavern_helper / 平台级自研引擎）：不转，仅识别 + 报告。

资产治理：

- 跨版本兼容：当卡片用到较新原语而当前版本尚不支持时，退化为文本或占位，保证不崩溃。
- 资产去重：通过语义标签与相似度匹配，避免模板重复堆积。

## 长期方向

正式版后可能继续推进：

- 知识库
- 破限库
- Prompt 路由器
- 智能记忆系统
- 模糊回忆
- 背包 / 数值结算等独立玩法系统（与积木 UI 解耦，单独立项）
- 更丰富的角色状态与剧情状态管理

---

## 当前原则

- 先保证本地工具稳定
- 不托管用户内容
- 不内置官方社区
- 资产分享采用文件流通
- 导入时严格检查内部标识
- 尽量兼容旧版本资产格式
