# Roadmap

本文档记录 LLM Project 当前阶段的开发计划。  
项目仍处于 Demo 阶段，规划内容可能根据实际体验、用户反馈和技术实现难度调整。

---

## 版本规则

### Demo 阶段

当前项目处于 Demo 阶段。

```text
Demo 1.x.x
```

用于构建正式版 `v1.0.0` 所需的基础功能。

### 正式版阶段

正式版发布后采用：

```text
vX.Y.Z
```

含义：

- `X.0.0`：某一大方向完成
- `X.Y.0`：通往下一个大方向的重要工程
- `X.Y.Z`：该工程内的优化、修复和细分版本

---

## Demo 1.3.x 总目标

Demo 1.3.x 的核心目标是：

```text
品牌确立 + 资产体系稳定 + 设定系统稳定化
```

让软件从一个功能 Demo，逐步过渡为有明确名称、稳定资产格式和较完整角色设定能力的可传播工具。

---

## Demo 1.3.0：品牌确立版

优先级：P0

计划内容：

- 正式确定软件名称
- 确定应用图标
- 统一 App 显示名
- 最终确认包名 / namespace / MainActivity package
- 统一 README、Release、关于页中的品牌文案
- 统一导出目录名称
- 统一备份文件名前缀
- 规划旧 `LLM_PROJECT_*` 内部标识的兼容策略

说明：

Demo 1.3.0 是正式命名前的关键节点。该版本完成后，后续功能将以正式名称继续推进。

---

## Demo 1.3.1：资产系统稳定版

优先级：P1

计划内容：

- 稳定完整备份 `.llmbak`
- 稳定角色卡文件包 `.llmcard`
- 稳定图片角色卡 `.llmchar.png`
- 稳定世界书 `.llmworld.json`
- 稳定背景卡 `.llmbg.*`
- 统一资产导入预览弹窗
- 统一导入错误提示
- 统一内部资产标识校验
- 优化资产导入导出交互

目标：

让用户可以安全地备份、迁移、分享和导入资产。

---

## Demo 1.3.2：状态栏基础版

优先级：P2

计划内容：

- 实装聊天页顶部状态栏
- 新增角色当前状态数据
- 支持好感度
- 支持心情
- 支持时间
- 支持地点
- 支持关系阶段
- 支持状态备注
- 点击状态栏打开状态编辑面板
- 将当前状态注入 Prompt

目标：

让状态栏从 UI 占位变成真实可用的角色状态系统。

---

## Demo 1.3.3：Prompt 分频注入版

优先级：P3

计划内容：

- 每轮注入核心身份提醒
- 小周期注入行为事实摘要
- 大周期注入完整详细设定
- 防止重要设定随长对话丢失
- 减少每轮重复注入完整设定造成的 token 开销
- 优化系统卡主角设定注入逻辑
- 优化角色设定、用户设定、世界书之间的去重规则

目标：

在角色稳定性和 token 开销之间取得平衡。

---

## Demo 1.3.4：角色设定版本管理基础版

优先级：P4

计划内容：

- 手动保存角色设定版本
- 支持版本名称
- 支持版本备注
- 支持查看版本列表
- 支持恢复指定版本
- 为后续自动设定更新做基础

目标：

让角色设定可以被保存、回退和管理，避免设定修改不可逆。

---

## Demo 1.3.5：自动设定维护初版

优先级：P5

计划内容：

- LLM 定期总结对话
- 自动生成新版角色设定草稿
- 启用但为空的条目可由 LLM 自动填充
- 自动设定更新先保存为草稿
- 用户确认后再应用
- 初步支持设定版本回退

目标：

让 LLM 辅助维护角色设定，但避免自动覆盖用户原始设定。

---

## Demo 1.3.6：世界书高级触发版

优先级：P6

计划内容：

- 世界书触发概率
- Sticky 粘性触发
- Cooldown 冷却
- Delay 延迟触发
- 优化世界书运行时状态
- 减少重复触发和刷屏

目标：

让世界书从关键词匹配升级为更灵活的设定激活系统。

---

## Demo 1.3.7：输入功能区实用版

优先级：P7

计划内容：

- 完善输入框功能区
- 添加继续回复按钮
- 添加重新生成按钮
- 添加状态入口
- 添加旁白模板
- 添加动作模板
- 添加骰子 / 随机判定入口

目标：

让输入框功能区从占位按钮变成实用工具区。

---

## Demo 1.3.8：角色心声版

优先级：P8

计划内容：

- 添加角色心声 / 内心独白功能
- 支持手动生成当前角色心声
- 支持特殊心声气泡显示
- 可选择是否保存到聊天记录

说明：

该功能不是展示模型真实推理链，而是生成角色扮演意义上的内心独白。

## 长期方向

正式版后可能继续推进：

- 知识库
- 破限库
- Prompt 路由器
- 智能记忆系统
- 模糊回忆
- 声明式 UI 积木系统
- 更丰富的角色状态与剧情状态管理

---

## 当前原则

- 先保证本地工具稳定
- 不托管用户内容
- 不内置官方社区
- 资产分享采用文件流通
- 导入时严格检查内部标识
- 尽量兼容旧版本资产格式
