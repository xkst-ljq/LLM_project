# 使用说明

本文档适用于 LLM Project Demo 1.2.5。  
当前项目仍处于 Demo 阶段，部分功能和界面可能在后续版本中调整。

---

## 1. 安装与首次启动

下载 APK 后直接安装。  
如果系统提示未知来源应用，请在系统设置中允许当前浏览器或文件管理器安装应用。

首次启动后，建议先完成：

```text
1. 配置 API
2. 设置全局用户信息
3. 创建或导入角色卡
```

---

## 2. API 配置

进入：

```text
设置 → API 配置
```

点击添加配置后填写：

```text
配置名称
Base URL
API Key
模型名
```

填写完成后可以点击测试连接。

### 常见 Base URL 示例

DeepSeek：

```text
https://api.deepseek.com
```

模型名示例：

```text
deepseek-chat
```

不同服务商的 Base URL 和模型名请以对应服务商文档为准。

### API Key 保存位置

API Key 使用 `flutter_secure_storage` 保存在本地设备。  
开发者不会收集、上传或托管用户 API Key。

---

## 3. 全局用户设定

进入：

```text
设置 → 用户设定
```

可以设置：

```text
用户昵称
用户头像
```

这个设定会作为默认用户信息。  
当角色卡没有单独用户覆盖设定，且系统卡没有启用主角设定时，会自动使用这里的全局用户设定。

---

## 4. 角色库

进入角色库后，可以看到所有角色卡。

### 新建或导入

点击右上角 `+`：

```text
新建角色卡
导入角色卡
```

### 选中角色卡

点击角色卡会选中它。  
再次点击选中的角色卡会进入编辑页。

### 进入聊天

选中角色卡后，点击右下角播放按钮进入聊天页。

### 导出角色卡

选中角色卡后点击导出按钮，可以选择：

```text
导出完整角色卡文件
导出角色卡图片
```

#### 完整角色卡文件

格式：

```text
.llmcard
```

这是稳定分享和迁移推荐格式。

#### 图片角色卡

格式：

```text
.llmchar.png
```

适合展示分享。  
通过聊天软件发送时请使用“原图”或“文件”方式，否则内部数据可能丢失。

---

## 5. 角色卡编辑

角色卡编辑页可以设置：

```text
角色名称
简短描述
角色头像
角色卡封面
卡片类型
绑定世界书
角色设定条目
开场白
```

### 人物卡

适合普通角色扮演对象。

常见条目包括：

```text
名称
与用户关系
身体数据
心理数据
背景数据
```

### 系统卡

适合世界、游戏、剧本、系统流等设定。

常见条目包括：

```text
系统名称
系统概要
系统详情
主角设定
剧情
```

如果系统卡启用了主角设定，则聊天时可自动使用主角设定作为用户默认设定。

---

## 6. 开场白

角色卡编辑页中可以添加开场白。

如果当前角色没有任何聊天历史，进入聊天页时会自动插入第一条非空开场白。

如果已有聊天记录，后续新增或修改开场白不会自动插入到已有对话中。  
如需重新显示开场白，可以清空聊天历史。

---

## 7. 世界书

世界书用于补充世界观、地点、人物、事件、规则等设定。

### 世界书条目

每个条目可以设置：

```text
标题
关键词
内容
是否常驻
是否递归触发
```

### 触发逻辑

- 常驻条目会持续生效
- 关键词条目会根据最近对话触发
- 递归触发可以让已触发内容继续激活其他条目

### 导入 / 导出

世界书可以导出为：

```text
.llmworld.json
```

导入时会先显示预览，确认后才真正导入。

---

## 8. 背景图库

背景图库支持：

```text
图片背景
纯色背景
渐变背景
```

图片背景可以导出为背景卡。  
背景卡保持原图比例，不会强制裁剪成 2:3。

导出位置：

```text
Download / LLM Project / Backgrounds
```

---

## 9. 聊天页

聊天页包含：

```text
顶部状态栏
聊天消息列表
底部输入区
角色切换入口
右侧设置面板
```

### 角色切换

底部角色名胶囊可点击打开角色切换轮盘。  
选择角色后可查看角色详情并切换当前聊天角色。

### 消息操作

AI 消息支持：

```text
重新生成
继续回复
开场白切换
多版本切换
```

用户消息支持：

```text
点击编辑
删除最近用户消息及后续回复
```

---

## 10. 当前卡用户设定

在聊天页右侧设置面板中进入：

```text
用户设定
```

这里设置的是当前角色卡的用户覆盖设定。

优先级为：

```text
当前卡用户覆盖设定
> 系统卡主角设定
> 全局用户设定
```

也就是说：

- 如果当前卡保存了用户覆盖设定，就使用覆盖设定
- 如果没有覆盖设定，但系统卡启用了主角设定，就使用主角设定
- 如果都没有，就使用全局用户设定

---

## 11. 备份与恢复

进入：

```text
设置 → 备份与恢复
```

可以导出完整备份：

```text
.llmbak
```

导出内容可以自定义，包括：

```text
角色卡
世界书
背景图库
图片资源
全局用户设定
当前角色用户覆盖设定
聊天记录
API 配置
API Key
排序和界面偏好
```

默认不导出：

```text
API Key
聊天记录
当前角色用户覆盖设定
```

因为这些可能包含隐私。

导出位置：

```text
Download / LLM Project / Backups
```

---

## 12. 资产导入说明

导入时不依赖文件扩展名，而是读取文件内部标识。  
如果文件不是 LLM Project 导出的资产，会提示无法识别。

### 常见导入失败原因

```text
文件不是 LLM Project 资产
文件被聊天软件压缩
图片不是原图
文件数据损坏
资产版本高于当前 App 支持版本
```

如果是图片角色卡或背景卡，请确认发送方使用了：

```text
原图发送
或
文件发送
```

## 13. 常见问题

### Q：为什么 API 连接失败？

请检查：

```text
Base URL 是否正确
API Key 是否正确
模型名是否正确
手机网络是否正常
App 是否允许联网
```

如果错误是：

```text
401 / 403
```

通常是 API Key 问题。

如果错误是：

```text
Failed host lookup
```

通常是网络、DNS 或联网权限问题。

---

### Q：为什么小米 / 澎湃 OS 顶部有黑框？

请在系统设置中找到本应用，将状态栏 / 刘海显示设置为：

```text
始终显示
```

---

### Q：为什么图片角色卡导入失败？

可能是图片被聊天软件压缩，导致内部数据丢失。  
请让对方选择：

```text
原图
或
以文件方式发送
```

---

### Q：为什么开场白没有显示？

开场白只会在当前角色没有任何聊天历史时自动插入。  
如果已有聊天记录，新增开场白不会插入到对话中间。

---

### Q：备份可以迁移到改包名后的版本吗？

可以。  
完整备份会导出数据库数据和图片资源，适合包名变化、换手机、重装 App 时迁移。
