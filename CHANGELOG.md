# Changelog

本文档记录 LLM Project 的主要版本变化。

---

## Unreleased（开发中）

### Added

#### 角色扩展元信息（补齐与酒馆卡的差距 · 批1）

- 新增 `CharacterMeta` 模型与 `characters.meta_json` 列（数据库 v2 → v3 安全迁移）
- 统一承载标签 / 作者 / 作者备注 / 角色版本 / 来源格式 /
  历史后注入指令(post_history_instructions) / 示例对话(mes_example)
- 角色详情面板展示「角色信息」（标签 / 作者 / 版本 / 来源 / 作者备注）
- `.llmcard` 导出 / 导入、完整备份均携带 `meta_json`
- 角色卡转换工具现在会写入 `meta_json`（标签 / 作者 / 来源等不再仅靠条目承载）

> 说明：元信息默认不注入 Prompt，仅用于展示与保留。标签筛选、
> 历史后注入位、示例对话注入将在后续批次开放。

#### 角色信息编辑与展示（批2）

- 角色编辑页新增「角色信息」编辑区：标签（逗号分隔）、作者、版本、作者备注
- 编辑页保存时写入 `meta_json`（保留转换器带来的 source_format / post_history 等字段）
- 角色库卡片展开时显示标签（最多 4 个）
- 这些信息手动新建的角色也能填写使用，不依赖转换器

> 标签筛选（角色库顶部筛选栏）将在批2 后续小步加入。

#### 第三方角色卡转换工具（005-A，规则转换）

- 新增纯 Dart 转换核心 `lib/tools/character_converter/`（PC 与移动端共用）
- 支持识别 PNG 内嵌角色卡（tEXt / zTXt / iTXt 的 `chara` / `ccv3`）
- 支持 SillyTavern / Character Card V2 与 TavernAI / V1 JSON
- 字段映射到 LLM Project 角色结构（名称 / 描述 / 性格 / 场景 / 示例对话 /
  开场白 / 备用开场白 / 系统提示 / 后置指令 / 来源信息 / 作者备注）
- 内嵌 `character_book` 转换为 LLM Project 世界书并自动绑定角色
- 世界书高级触发规则（优先级 / 位置 / 深度 / 概率等）降级并在报告中提示
- 完整保留原始第三方 JSON（禁用条目），便于后续重新转换
- 输出标准 `.llmcard`（可被「导入角色卡」直接识别）+ 转换报告（json/txt）
- 桌面端支持拖入文件 / 文件夹批量转换，移动端支持多选文件
- 设置页新增「角色卡转换工具」入口
- 新增单元测试 `test/character_converter_test.dart`

> 不执行酒馆脚本 / 正则 / 扩展插件，只做解析、映射与保留。
> AI 智能归类与精修对照为后续增强（005-B/C）。
